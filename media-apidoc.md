# Media & Agent API Documentation

This documentation covers the newly implemented API endpoints for the Media and AI Agent features in the backend.

All endpoints require authentication via JWT token.

## Base URL
`/api`

## Authentication
Add the following header to all requests:
```http
Authorization: Bearer <your_access_token>
```

---

## 1. Document Management Endpoints

### 1.1. Get All Documents
Retrieve a list of all documents uploaded to the server.

*   **URL:** `/documents`
*   **Method:** `GET`
*   **Headers:** `Authorization: Bearer <token>`
*   **Success Response:**
    *   **Code:** `200 OK`
    *   **Content:**
        ```json
        [
          {
            "id": "uuid",
            "title": "Workshop_day1_AICOE",
            "file_url": "/uploads/filename.pdf",
            "file_size": 2048000,
            "file_type": "application/pdf",
            "location": "Olah Rasio Room",
            "created_at": "2026-05-29T10:00:00.000Z",
            "updated_at": "2026-05-29T10:00:00.000Z",
            "modifiedBy": {
              "id": "user-uuid",
              "name": "User Name",
              "avatar": "..."
            }
          }
        ]
        ```

### 1.2. Upload Document
Upload a new document/media file.

*   **URL:** `/documents/upload`
*   **Method:** `POST`
*   **Headers:** `Authorization: Bearer <token>`
*   **Content-Type:** `multipart/form-data`
*   **Body:**
    *   `file` (File, Required): The file to be uploaded.
    *   `title` (String, Optional): The title of the document. Falls back to original filename if not provided.
    *   `location` (String, Optional): The location context for the document.
*   **Success Response:**
    *   **Code:** `201 Created`
    *   **Content:**
        ```json
        {
          "id": "uuid",
          "title": "My Uploaded File",
          "file_url": "/uploads/123456789.pdf",
          "file_size": 102400,
          "file_type": "application/pdf",
          "location": "Server Upload",
          "created_at": "2026-05-29T10:00:00.000Z",
          "updated_at": "2026-05-29T10:00:00.000Z",
          "modifiedBy": {
            "id": "user-uuid"
          }
        }
        ```

### 1.3. Delete Document
Delete an existing document by ID.

*   **URL:** `/documents/:id`
*   **Method:** `DELETE`
*   **Headers:** `Authorization: Bearer <token>`
*   **URL Params:** `id` (UUID of the document)
*   **Success Response:**
    *   **Code:** `200 OK`
    *   **Content:**
        ```json
        {
          "id": "uuid",
          "title": "Deleted Document",
          "...": "..."
        }
        ```

## 2. AI Document Generation Endpoints

These endpoints are used to generate specific content based on source documents and context provided by the user.

### 2.1. Generate Recap
Generate a recap based on provided context and documents.

*   **URL:** `/documents/generate/recap`
*   **Method:** `POST`
*   **Headers:** `Authorization: Bearer <token>`, `Content-Type: application/json`
*   **Body:**
    ```json
    {
      "source_document_ids": ["uuid1", "uuid2"],
      "context": "Minta tolong buatkan rekap dari presentasi narasumber"
    }
    ```
*   **Success Response:**
    *   **Code:** `201 Created`
    *   **Content:**
        ```json
        {
          "result": "Mocked recap generated by AI based on context: ...",
          "document_url": "http://mockurl.com/recap.pdf"
        }
        ```

### 2.2. Generate Report
Generate a formal report based on provided context and documents.

*   **URL:** `/documents/generate/report`
*   **Method:** `POST`
*   **Headers:** `Authorization: Bearer <token>`, `Content-Type: application/json`
*   **Body:**
    ```json
    {
      "source_document_ids": ["uuid1"],
      "context": "Buatkan laporan kegiatan hari ini"
    }
    ```
*   **Success Response:**
    *   **Code:** `201 Created`
    *   **Content:**
        ```json
        {
          "result": "Mocked report generated by AI based on context: ...",
          "document_url": "http://mockurl.com/report.pdf"
        }
        ```

### 2.3. Generate Minutes of Meeting (MoM)
Generate MoM based on discussion recordings or transcripts.

*   **URL:** `/documents/generate/mom`
*   **Method:** `POST`
*   **Headers:** `Authorization: Bearer <token>`, `Content-Type: application/json`

    *   **Code:** `201 Created`
    *   **Content:**
        ```json
        {
          "result": "Mocked MoM generated by AI based on context: ...",

        }
        ```

## 3. AI Agent (Ask ChatAja Agent)

Endpoint for interacting with the smart AI assistant to query information.

### 3.1. Ask Agent
Ask the AI Agent a question or issue a command.

*   **URL:** `/agent/chat`
*   **Method:** `POST`
*   **Headers:** `Authorization: Bearer <token>`, `Content-Type: application/json`
*   **Body:**
    ```json
    {
      "query": "Tolong cari dokumen tentang rekap kegiatan bulan lalu"
    }
    ```
*   **Success Response:**
    *   **Code:** `201 Created`
    *   **Content:**
        ```json
        {
          "answer": "This is a mocked AI response for the query: \"Tolong cari dokumen tentang rekap kegiatan bulan lalu\"",
          "references": []
        }
        ```
